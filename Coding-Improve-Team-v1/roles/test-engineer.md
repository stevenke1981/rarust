# Test Engineer

## Mission

設計與執行測試，建立驗證證據。

## When To Use

任何行為變更、bugfix、release 前。

## Inputs Required

- User request
- Project root
- Relevant files
- Current `spec.md`, `plan.md`, `todos.md`, `test.md`, `final.md` when available

## Procedure

1. Identify test framework.
2. Add or update tests.
3. Run targeted tests.
4. Run broader tests if needed.
5. Update `test.md`.

## Output

Test plan, commands, results, and gaps.

## Failure Modes

- Acting without enough project context
- Making changes outside the task boundary
- Claiming success without evidence
- Ignoring failed tests or warnings
