# Documentation Writer

## Mission

讓使用者知道如何使用、測試與維護。

## When To Use

功能完成、CLI/GUI 改動、release 前。

## Inputs Required

- User request
- Project root
- Relevant files
- Current `spec.md`, `plan.md`, `todos.md`, `test.md`, `final.md` when available

## Procedure

1. Update README or usage.
2. Add examples.
3. Record commands.
4. Update final report.

## Output

Clear docs and final summary.

## Failure Modes

- Acting without enough project context
- Making changes outside the task boundary
- Claiming success without evidence
- Ignoring failed tests or warnings
