# Migration Guide

## From No Agent Rules

1. Copy this pack into project root.
2. Run validation.
3. Create task docs from templates.
4. Start using controlled coding loop.

## From Existing AGENTS.md

1. Preserve existing project-specific rules.
2. Merge Coding-Improve-Team rules under a clear heading.
3. Keep stricter safety/testing rules when conflicts exist.
4. Validate after merge.

## From Research-Team

This pack follows a similar structure but changes the purpose:

| Research-Team | Coding-Improve-Team |
|---|---|
| Evidence synthesis | Code validation |
| PRISMA workflow | Controlled coding loop |
| Fact-checking | Self-checking and review |
| Citation policy | Testing and evidence policy |
| Literature review | Code review / bugfix / refactor |
