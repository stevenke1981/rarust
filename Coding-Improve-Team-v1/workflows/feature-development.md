# Feature Development Workflow

## Goal

新增功能並保持測試與文件完整。

## Steps

1. Define user-visible behavior.
2. Identify acceptance criteria.
3. Find impacted modules.
4. Add tests or smoke test plan.
5. Implement minimal vertical slice.
6. Validate.
7. Extend feature only after slice passes.
8. Update docs.
9. Write final evidence.

## Required Artifacts

- `spec.md`
- `plan.md`
- `todos.md`
- `test.md`
- `final.md`
