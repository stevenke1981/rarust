# Test-Driven Development Workflow

## Goal

用測試約束行為，降低 agent 寫錯的機率。

## Steps

1. Define expected behavior.
2. Write failing test or smoke test.
3. Implement minimal code to pass.
4. Refactor safely.
5. Run full relevant tests.
6. Document test evidence.

## If Tests Are Hard

Use one of:

- CLI smoke test
- Golden output comparison
- Snapshot test
- Manual run script
- Type check / build check
