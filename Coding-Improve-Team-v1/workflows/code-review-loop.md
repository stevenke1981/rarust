# Code Review Loop

## Goal

在完成任務前審查 diff。

## Review Questions

1. Does the change satisfy the spec?
2. Are edge cases handled?
3. Are tests adequate?
4. Are errors handled clearly?
5. Does the change introduce security risks?
6. Is the diff minimal?
7. Is documentation updated?

## Decision

- Approve
- Approve with notes
- Request changes
- Block
