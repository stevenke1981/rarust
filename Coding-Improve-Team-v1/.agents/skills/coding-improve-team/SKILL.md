# Coding Improve Team Skill

Use this skill when the user asks to improve code, build a feature, fix a bug, refactor, add tests, review code, prepare release, or run a self-checking coding workflow.

## Core Rule

Do not start by editing code. Start by understanding the project boundary and writing or updating:

- `spec.md`
- `plan.md`
- `todos.md`
- `test.md`
- `final.md`

## Default Workflow

1. Identify project root.
2. Read project docs and build files.
3. Determine task type.
4. Write acceptance criteria.
5. Plan small steps.
6. Implement one step.
7. Run validation.
8. Review diff.
9. Fix or rollback.
10. Record final evidence and lessons.

## Use These Documents

- `AGENTS.md` for global agent rules.
- `TEAM.md` for role charter.
- `workflows/controlled-coding-loop.md` for standard execution.
- `workflows/failure-recovery.md` after repeated failure.
- `policies/project-boundaries.md` to avoid mixing projects.
- `policies/evidence-and-testing.md` before claiming completion.

## Required Final Answer

Always include:

- What changed.
- What was validated.
- What could not be validated.
- Remaining risks.
- Next suggested step.
