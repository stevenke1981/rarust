# /triage

Purpose: 對 bug、需求或混亂專案做初步分類。

## Procedure

1. Identify current project type.
2. Identify build system.
3. Identify test system.
4. Identify top risks.
5. Classify task type:
   - Feature
   - Bugfix
   - Refactor
   - Test
   - Documentation
   - Release
6. Recommend the next command.

## Output

A concise triage report with next action.
