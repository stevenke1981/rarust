# /review

Purpose: 審查目前 diff 或最近修改。

## Review Dimensions

- Requirement fit
- Correctness
- Edge cases
- Test coverage
- Security
- Performance
- Maintainability
- Documentation
- Rollback risk

## Output Format

1. Summary
2. Blocking Issues
3. Non-blocking Suggestions
4. Test Gaps
5. Risk Rating
6. Approval Decision
